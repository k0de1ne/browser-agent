# AI Browser Agent

Автономный AI-агент для автоматизации веб-браузера, способный выполнять сложные многошаговые задачи без предварительного программирования сценариев.

## 🎯 О проекте

Этот проект был создан в рамках тестового задания на позицию AI-инженера. Агент демонстрирует способность автономно управлять браузером, принимать решения в реальном времени и адаптироваться к неожиданным ситуациям.

### Что умеет агент

- **Полностью автономная работа**: Агент самостоятельно планирует действия и адаптирует план по ходу выполнения
- **Динамическое взаимодействие**: Находит элементы на странице без предопределенных селекторов
- **Контекстное понимание**: Использует семантический анализ страницы для принятия решений
- **Security layer**: Запрашивает подтверждение перед деструктивными действиями
- **Умное управление контекстом**: Экстракция только релевантной информации вместо отправки всего DOM

## ✅ Реализованные требования

### Автоматизация браузера
- ✅ Программное управление через Playwright
- ✅ Persistent sessions (возможность войти вручную и продолжить работу с агентом)
- ✅ Видимый браузер (не headless) с возможностью наблюдать за действиями

### Автономный AI-агент
- ✅ Использует OpenAI API (совместимо с Claude)
- ✅ Работает полностью автономно без постоянного участия пользователя
- ✅ Обрабатывает многошаговые задачи с навигацией между страницами
- ✅ Динамическое планирование (создает и адаптирует план выполнения в процессе работы)

### Управление контекстом
- ✅ Интеллектуальная экстракция DOM (только интерактивные и семантические элементы)
- ✅ Приоритизация видимых элементов в viewport
- ✅ Фильтрация по тексту и селекторам для уменьшения размера контекста
- ✅ Семантический анализ структуры страницы (landmarks, headings, navigation)

### Продвинутые паттерны
- ✅ **Security layer**: Детектирует деструктивные действия (оплата, удаление, изменение настроек) и запрашивает подтверждение
- ✅ **Обработка ошибок**: Агент адаптируется при неудачных действиях и пробует альтернативные подходы
- 🔄 **Sub-agent architecture**: Базовая архитектура готова, но требует дальнейшей разработки (см. "Планы развития")

### Никаких жестко заданных сценариев
- ✅ Агент динамически определяет следующий шаг на основе анализа страницы
- ✅ Нет предопределенных селекторов - агент сам находит нужные элементы
- ✅ Нет хардкода URL или текстов кнопок - агент исследует страницу самостоятельно

### Ключевые технические решения

#### 1. Интеллектуальная экстракция DOM
Вместо отправки всего HTML в LLM, система извлекает только релевантную информацию:

- **Фильтрация**: Только интерактивные элементы (кнопки, ссылки, поля ввода) и семантические маркеры (заголовки, навигация)
- **Приоритизация**: Элементы в viewport показываются первыми
- **Семантический контекст**: Каждый элемент аннотируется контекстом (например, "in navigation", "under heading 'Checkout'")
- **Структурированный вывод**: Элементы группируются по видимости и представляются в удобном формате

**Пример вывода для LLM:**
```
=== VISIBLE IN VIEWPORT ===
[el-1] button "Add to Cart" [type="submit"] {in main content, under "Product Details"} @(450px)
[el-2] input "" [type="text", placeholder="Search products"] {in navigation} @(20px)

=== BELOW VIEWPORT (need scroll) ===
[el-15] button "Buy Now" [type="submit"] {in main content} @(1200px)
```

#### 2. Security Layer
Многоуровневая система защиты от деструктивных действий:

- **Детекция по ключевым словам**: Анализ текста элементов на нескольких языках (delete, удалить, pay, оплатить)
- **Анализ атрибутов**: Проверка типов форм (payment, card, cvv)
- **URL patterns**: Детекция чувствительных страниц (checkout, settings, admin)
- **Семантический контекст**: Понимание контекста действия
- **Уровни риска**: low → medium → high → critical с автоматическим запросом подтверждения

#### 3. Динамическое планирование
Агент создает и адаптирует план по ходу выполнения:

```typescript
// Агент создает план в начале задачи
update_plan({
  steps: [
    {description: "Navigate to site", status: "pending"},
    {description: "Find search", status: "pending"}
  ],
  current_step_index: 0,
  adaptations: []
})

// По ходу работы агент обновляет план
update_plan({
  steps: [...previous_steps, 
    {description: "Handle popup", status: "completed"}, // новый шаг
    {description: "Find search", status: "in_progress"}
  ],
  adaptations: ["Added step to handle unexpected popup"]
})
```

#### 4. Tool-based архитектура
Агент использует набор инструментов для взаимодействия:

**Навигация:**
- `navigate(url)` - переход на страницу
- `go_back()` - вернуться назад
- `scroll(direction)` - прокрутка страницы

**Исследование:**
- `get_page_structure()` - структура страницы (landmarks, headings)
- `get_page_content(text_contains?, selector?, max_elements?)` - интерактивные элементы
- `find_element(selector?, text?, attribute?)` - точный поиск элемента
- `read_page_text(selector?, max_length?)` - чтение текстового контента

**Взаимодействие:**
- `click(element_id)` - клик по элементу
- `type_text(element_id, text, press_enter?)` - ввод текста

**Управление:**
- `update_plan(steps, current_step_index, adaptations)` - обновление плана
- `ask_user(question, reason)` - запрос помощи пользователя
- `complete_task(summary, success)` - завершение задачи

## 🚀 Быстрый старт

### Требования
- Node.js 18+
- npm или yarn

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/k0de1ne/browser-agent
cd browser-agent

# Установить зависимости
npm install

# Настроить переменные окружения
cp .env.example .env
# Отредактировать .env и добавить API ключ
```

### Конфигурация

Отредактируйте `.env` файл:

```bash
# Для OpenAI
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4-turbo-preview
LLM_API_KEY=sk-your-api-key-here

# Для локального LLM (например, LM Studio)
LLM_BASE_URL=http://127.0.0.1:1234/v1
LLM_MODEL=mistralai/ministral-3-14b-reasoning
LLM_API_KEY=not-needed

# Настройки браузера
HEADLESS=false
BROWSER_TIMEOUT=30000
```

### Запуск

```bash
# Development режим
npm run dev

# Production режим
npm run build
npm start
```

## 🔍 Как это работает

### Жизненный цикл задачи

1. **Инициализация**
   - Пользователь вводит задачу
   - Агент создает начальный план

2. **Исследование**
   - Агент анализирует структуру страницы (`get_page_structure`)
   - Извлекает интерактивные элементы (`get_page_content`)
   - Использует семантический контекст для понимания страницы

3. **Планирование действия**
   - На основе цели и текущего состояния страницы
   - Агент решает, какой инструмент использовать
   - Extended thinking (reasoning) помогает принять решение

4. **Выполнение**
   - Вызов инструмента (click, type_text, navigate)
   - Security layer проверяет действие
   - При необходимости - запрос подтверждения

5. **Верификация**
   - Проверка результата действия
   - Обновление плана
   - При ошибке - адаптация стратегии

6. **Итерация**
   - Цикл повторяется до завершения задачи
   - Максимум 50 итераций (настраивается)

### Пример работы

```
Task: Find Python jobs on hh.ru

Iteration 1:
🧠 Thinking: I need to navigate to hh.ru first
🔧 Tool: navigate
   Args: {"url": "https://hh.ru"}
✓  Result: Navigated to https://hh.ru

Iteration 2:
🧠 Thinking: Now I need to find the search field for jobs
🔧 Tool: get_page_content
   Args: {"text_contains": "search", "max_elements": 30}
✓  Result: Found 15 elements (12 visible, 3 below viewport)

Iteration 3:
🧠 Thinking: I found the search input [el-3], let me type "Python"
🔧 Tool: type_text
   Args: {"element_id": "el-3", "text": "Python", "press_enter": true}
✓  Result: Typed "Python" and pressed Enter

... (продолжение до завершения)
```

## 🛡️ Security Layer в действии

Когда агент пытается выполнить потенциально опасное действие:

```
════════════════════════════════════════════════════════════
🔴 SECURITY CHECK [FINANCIAL]
Risk: HIGH
Reason: Detected financial keyword: "pay" (en)

Element: "Complete Purchase"
Page: https://example.com/checkout

Do you want to proceed with this action?
════════════════════════════════════════════════════════════
? Proceed? (y/N)
```

Система детектирует:
- 💳 Финансовые операции (payment, checkout, buy)
- 🗑️ Удаление данных (delete, remove)
- ⚙️ Изменение настроек (settings, configure)
- 🔐 Действия с аккаунтом (logout, deactivate)
- 📧 Отправка данных (send, submit payment forms)

## 📈 Планы развития

Этот проект был создан за ограниченное время для тестового задания, но я вижу в нем большой интерес и хотел бы закончить проект:

### Приоритет 1: Архитектура памяти
- [ ] **Vector Database для долговременной памяти**
  - Сохранение успешных паттернов решения задач
  - Семантический поиск похожих ситуаций
  - Рассматриваю: Chroma, qdrant или самописная in-memory KV база
  
- [ ] **In-memory KV хранилище для сессии**
  - Кэширование структуры посещенных страниц
  - История успешных селекторов
  - Уменьшение повторных запросов к DOM

### Приоритет 2: Улучшение инструментов
- [ ] **Интеграция Playwright MCP**
  - Использовать официальный Playwright MCP вместо самописных инструментов
  - Доступ к продвинутым возможностям (network interception, мокирование)

### Приоритет 3: Параллелизм и Sub-agents
- [ ] **Параллельное выполнение инструментов**
  - Одновременный анализ нескольких вкладок
  - Асинхронная обработка независимых действий
  
- [ ] **Динамическая система Sub-agents**
  - Специализированные агенты для разных доменов (e-commerce, форумы, документация)
  - Агент-координатор для распределения задач
  - Обмен контекстом между агентами

### Приоритет 4: Оптимизация токенов
- [ ] **Компрессия контекста**
  - Суммаризация истории действий
  - Удаление нерелевантной информации из контекста
  
- [ ] **Адаптивный выбор модели**
  - Легкие модели для простых действий
  - Тяжелые модели только для сложных решений

### Приоритет 5: Robustness
- [ ] **Recovery механизмы**
  - Автоматическое восстановление после краша браузера
  - Retry логика с exponential backoff
  
- [ ] **Тестирование**
  - Unit тесты для критических компонентов
  - Integration тесты на реальных сайтах
  - Benchmark для популярных задач

### Приоритет 6: Пользовательский опыт
- [ ] **Web UI вместо CLI**
  - Использовать продвинутые cli инстументы

- [ ] **Streaming responses**
  - Показ мыслительного процесса в реальном времени
