# AI Browser Agent

Автономный AI-агент для автоматизации веб-браузера. Выполняет сложные многошаговые задачи без предварительного программирования сценариев, самостоятельно исследуя страницы и принимая решения в реальном времени.

## О проекте

Проект создан в рамках тестового задания. Агент получает текстовую задачу от пользователя и автономно выполняет её в браузере:

- Открывает сайты и исследует их структуру
- Находит нужные элементы без хардкода селекторов
- Заполняет формы, кликает по кнопкам, навигируется между страницами
- Адаптируется к ошибкам и неожиданным ситуациям
- Запрашивает подтверждение перед опасными действиями (оплата, удаление)

### Примеры задач

```
Find Python developer jobs in Moscow on hh.ru
```

```
Go to Amazon, search for "wireless headphones", filter by 4+ stars
```

```
Delete all spam emails from Gmail inbox
```

## Быстрый старт

### Установка

```bash
git clone https://github.com/zqdma/browser-agent
cd browser-agent
npm install
```

### Конфигурация

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```bash
# OpenAI API
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4-turbo-preview
LLM_API_KEY=sk-your-key

# Или локальная модель (LM Studio)
# LLM_BASE_URL=http://127.0.0.1:1234/v1
# LLM_MODEL=mistralai/ministral-3-14b-reasoning
# LLM_API_KEY=not-needed

HEADLESS=false
BROWSER_TIMEOUT=30000
```

### Запуск

```bash
# Development
npm run dev

# Production
npm run build && npm start
```

## Реализованные требования

### Автоматизация браузера
- Программное управление через Playwright
- Persistent sessions - можно войти вручную, агент продолжит работу
- Видимый браузер (не headless)

### Автономный AI-агент
- Полностью автономная работа без постоянного участия пользователя
- Многошаговые задачи с навигацией между страницами
- Динамическое планирование и адаптация

### Управление контекстом
- Интеллектуальная экстракция DOM - не весь HTML, а только релевантные элементы
- Приоритизация элементов в viewport
- Семантический анализ структуры страницы

### Security Layer
- Детекция опасных действий: оплата, удаление, изменение настроек
- Запрос подтверждения перед выполнением
- Анализ ключевых слов на нескольких языках (EN, RU, ES, DE, FR)

### Без хардкода
- Нет предопределенных селекторов
- Нет заготовленных URL
- Агент сам исследует страницу и принимает решения

## Архитектура

### Структура проекта

```
src/
├── agent/
│   ├── agent.ts          # Главный класс - оркестрация задач
│   ├── tools.ts          # инструменты для LLM
│   ├── security.ts       # Защита от деструктивных действий
│   └── visualizer.ts     # Визуализация в терминале
├── browser/
│   ├── manager.ts        # Управление браузером и вкладками
│   └── dom-extractor.ts  # Извлечение DOM для LLM
├── llm.ts                # Клиент OpenAI API
└── index.ts              # Точка входа
```

### Цикл работы агента

1. Пользователь вводит задачу
2. Агент создает план выполнения
3. Анализирует страницу через `get_page_content`
4. Выбирает действие (click, type, navigate)
5. Security layer проверяет безопасность
6. Выполняет действие
7. Повторяет до завершения задачи (макс. 50 итераций)

### Инструменты агента

**Навигация:** `navigate`, `go_back`, `scroll`

**Исследование:** `get_page_structure`, `get_page_content`, `find_element`, `read_page_text`

**Взаимодействие:** `click`, `type_text`, `wait`

**Вкладки:** `new_tab`, `switch_tab`, `list_tabs`, `close_tab`

**Управление:** `update_plan`, `ask_user`, `complete_task`

## Экстракция DOM

Вместо отправки всего HTML в LLM, система извлекает только интерактивные элементы с семантическим контекстом:

```
=== VISIBLE IN VIEWPORT ===
[el-1] button "Add to Cart" [type="submit"] {in main content} @(450px)
[el-2] input "" [placeholder="Search"] {in navigation} @(20px)

=== BELOW VIEWPORT ===
[el-15] button "Buy Now" {in footer} @(1200px)
```

## Security Layer

При обнаружении потенциально опасного действия:

```
════════════════════════════════════════════════════════════
SECURITY CHECK [FINANCIAL]
Risk: HIGH
Reason: Detected financial keyword: "pay"

Element: "Complete Purchase"
Page: https://example.com/checkout

? Proceed? (y/N)
════════════════════════════════════════════════════════════
```

Категории риска:
- **Financial** - платежи, покупки
- **Data Loss** - удаление данных
- **Account** - изменение пароля, выход
- **Privacy** - настройки приватности
